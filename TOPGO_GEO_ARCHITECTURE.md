# TOPGO GEO 智能路由系统架构设计

## 一、系统概述

### 1.1 核心目标
打造能理解用户意图、感知网络生态、智能匹配最优AI资源的下一代API服务平台。

### 1.2 双重GEO定义

| 维度 | 定义 | 承担应用 |
|------|------|----------|
| **物理GEO** | 基于IP地理位置的网络层低延迟接入 | TOPGO Gateway |
| **逻辑GEO** | 基于语义、语言、合规的智能路由 | TOPGO Orchestrator |

## 三、TOPGO Gateway - 实时路由决策

### 3.1 请求上下文提取

```go
// RequestContext 请求上下文
type RequestContext struct {
    // 物理上下文
    ClientIP       string
    Country        string  // 国家
    City           string  // 城市
    ASN            string  // 运营商
    
    // 逻辑上下文
    ContentLanguage string  // 请求语言
    UserID          string  // 用户ID
    UserTier        string  // 用户套餐
    TargetModel     string  // 目标模型
    
    // 历史偏好
    HistoricalPref []string
}
```

### 3.2 路由决策引擎

```
┌─────────────────────────────────────────────────────────────┐
│                    GEO 路由决策引擎                          │
├─────────────────────────────────────────────────────────────┤
│  输入:                                                       │
│  ┌──────────────┐  ┌─────────────┐  ┌────────────────┐    │
│  │ 请求上下文    │  │ 资源池数据  │  │ 策略规则       │    │
│  │ (ClientIP,   │  │ (带标签的   │  │ (编排的路由    │    │
│  │  Language,  │→ │ 端点列表)   │→ │ 策略)          │    │
│  │  Model)      │  │             │  │                │    │
│  └──────────────┘  └─────────────┘  └────────────────┘    │
│                                                              │
│  处理:                                                       │
│  1. 上下文匹配 → 2. 综合评分 → 3. 排序选择                    │
│                                                              │
│  输出:                                                       │
│  ┌──────────────┐                                          │
│  │ 最优端点      │ → 转发请求                                │
│  │ (得分: 95)   │                                          │
│  └──────────────┘                                          │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 评分算法

```
Score = (策略优先级 × 0.3) + (健康分 × 0.25) + (延迟得分 × 0.25) + (负载权重 × 0.2)

其中:
- 策略优先级: 0-100 (根据匹配的策略规则)
- 健康分: 0-100 (来自Orchestrator的实时探测)
- 延迟得分: 100 - (实测延迟 / 基准延迟 × 100)
- 负载权重: 100 - (当前负载 / 最大负载 × 100)
```

## 五、实施路线图

### Phase 1: 基础物理GEO
- [ ] 集成 IP 地理库 (MaxMind GeoLite2)
- [ ] 实现基于区域的最短延迟路由
- [ ] 跨地域故障自动转移

### Phase 2: 逻辑GEO
- [ ] 资源多维度标签化体系
- [ ] 语言感知路由
- [ ] 合规性路由 (GDPR等)

### Phase 3: 智能GEO
- [ ] 智能策略编排引擎
- [ ] 动态资源画像与健康探测
- [ ] A/B 测试框架

### Phase 4: 高级特性
- [ ] 全球流量热力图可视化
- [ ] 机器学习优化的路由策略
- [ ] 自适应成本优化

## 六、技术选型

### IP地理库
- **MaxMind GeoLite2** - 免费、开源、精度高

### 健康探测
- **Prometheus + Blackbox Exporter** - 基础设施监控
- **自定义探测服务** - 多地域延迟测试

### 策略引擎
- **JSON Schema** - 规则定义
- **Redis** - 策略缓存与分发

### 可观测性
- **Grafana** - 可视化面板
- **ELK/ClickHouse** - 日志分析
